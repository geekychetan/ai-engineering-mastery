# Week 15 · Day 03 — Applied/case questions: framing problems out loud

**Estimated time:** ~5 hrs
**GitHub issue:** (link the Day-03 issue here)

## Objectives
- [ ] Practice applied "how would you build X?" case questions out loud.
- [ ] Lead with problem-framing, clarifying questions, and success metrics.
- [ ] Show judgment about the simplest thing that could work.

## Learn (with resources)
- **ByteByteGo** (YouTube) — requirement-gathering and tradeoff framing, adapted to LLM apps.
- Lean on the MBA/Business Analytics muscle: translate a fuzzy business problem into a measurable one.

## Homework (branch → PR)
1. [ ] Branch: `git switch -c week15-day03-applied-cases`.
2. [ ] In `notes/interview/case-questions.md`, list 5–6 realistic prompts, e.g. "Build a support-ticket triage assistant", "Build a doc Q&A for a legal team", "Build a tool that generates test cases from requirements" (your home turf).
3. [ ] For each, work a structured out-loud answer: **clarify** (who's the user, scale, constraints) → **define success + how you'd evaluate** → **simplest v1** (RAG? agent? plain prompt?) → **risks + mitigations** (hallucination, cost, latency, bad inputs) → **what you'd measure to iterate**.
4. [ ] Always name the eval plan — it's what separates you. Reference your capstone's golden-set + judge approach.
5. [ ] Record yourself answering 2 of them; note filler words, rambling, and whether you asked clarifying questions before designing.
6. [ ] Push, PR with `Closes #<issue>`, self-review, merge.
